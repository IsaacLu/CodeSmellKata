到這邊我們先想一下
什麼問題開始發生了?

1. 因為使用基礎型別，導致需要再所到之處需要驗證的地方都要再寫一次驗證行為
這就造成了 Duplicate Code

2. 如果有比較好的工程師發現了duplicate code, 他可能會把重複的邏輯抽出去
但是抽出去後也沒有一個規範，它可以有非常多選項:
如果是單一類別，他可以單純地用 local method
如果是多個類別有相同驗證邏輯就有
- Helper Class
- Extract Base Class
- Extention Method
- New Another Class
這些選項
然而只要有人沒有特別去找他，下一個人就可能會用另一種方式作驗證

3. 也因此在這邊推薦一個想法
第一個, 也就是解決基礎型別的方法，那就是先把相關聯的基礎型別抽成適合的物件
第二個, 讓驗證邏輯交給新抽出來適合的物件

這樣就可以讓驗證是相對容易被找到的
